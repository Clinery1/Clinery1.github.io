I am attending Florida Polytechnic University pursuing a degree in Computer Science. I have been programming since 2016, but I started with websites. I started learning actual programming languages about a year after that. The first programming language I learned was Rust. I am very glad I learned Rust first because it allowed me to learn the theory parts of programming without the headaches that other languages like C impose (looking at you pointers and array-out-of-bounds errors). I eventually learned more languages (see Languages I know) and started programming games.

▸ Personal mission statement
  • Any games I make where I am the sole creator AND/OR where I am LEADING a team of creators will:
    • NOT HAVE micro transactions
    • listen to user feedback to the extent that it does not alter the original vision of the game (NOTE: this includes any bug1 reports, difficulty changes, and UI/UX changes)

This list is subject to ADDITIONS, but NOT removals. See the git commit logs for verification of this statement.

1 A bug is any action that happens in a program that does something that is not expected. Example: the button labeled "Exit" brings you to the settings menu or the save button does nothing.
